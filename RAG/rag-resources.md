## What is RAG?

**Retrieval-Augmented Generation (RAG)** is a technique that combines the power of Large Language Models (LLMs) with external knowledge retrieval. Instead of relying solely on the LLM's training data, RAG systems retrieve relevant information from a knowledge base to generate more accurate, contextual, and up-to-date responses.

**Core Components:**
- **Retrieval:** Searching through a vector database to find relevant documents
- **Augmentation:** Adding retrieved context to the user's query
- **Generation:** Using an LLM to generate responses based on the augmented prompt

### Vector Databases Comparison

Choosing the right vector database is crucial for RAG:

| Database | Type | Best For | Pricing |
|----------|------|----------|---------|
| **Qdrant** | Open-source + Cloud | Fast, efficient, Rust-based | Free tier, $0.014/hr cloud |
| **Pinecone** | Managed Cloud | Easy setup, no DevOps | Free tier, $70+/month |
| **Weaviate** | Open-source + Cloud | GraphQL API, built-in vectorization | Free (self-hosted), $25+/month |
| **Milvus** | Open-source + Cloud | Massive scale (billions of vectors) | Free (self-hosted), cloud pricing varies |
| **Chroma** | Open-source | LLM app development | Free |

**Recommended Reading:** [Vector Database Comparison 2025](https://tensorblue.com/blog/vector-database-comparison-pinecone-weaviate-qdrant-milvus-2025)
